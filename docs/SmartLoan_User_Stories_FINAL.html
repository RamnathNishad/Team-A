2. Authentication Module

2.1 Customer Registration Page

CUST-FE-001
Customer Registration Form UI
Frontend
Register
Critical
5 pts

Description

As a prospective customer, I want to see a registration form where I can enter my details and create an account.

Acceptance Criteria

  • Display form fields: Email, Phone, Password, Confirm Password
  • Show password strength indicator (Weak/Medium/Strong)
  • Mark all fields as required with asterisk (*)
  • Display password requirements (Min 8 chars, 1 uppercase, 1 number, 1 special char)
  • Show real-time validation errors below each field
  • Disable Submit button until form is valid
  • Show "Already have account? Login" link
  • Responsive design for mobile (375px), tablet (768px), desktop (1920px)

Technical Requirements

  • Use React form library (React Hook Form)
  • Implement client-side validation using Zod
  • Email regex validation
  • Password regex with strength requirements
  • Store form state in React component state
  • Styling using Tailwind CSS with consistent spacing
CUST-BE-001
User Registration Endpoint
Backend
POST /api/v1/auth/register
Critical
5 pts

Description

As a backend system, I need to receive and process customer registration requests, validate data, and create user accounts.

Acceptance Criteria

  • Endpoint: POST /api/v1/auth/register
  • Validate all required fields (email, phone, password)
  • Check email uniqueness (return 400 if duplicate)
  • Check phone uniqueness (return 400 if duplicate)
  • Hash password using bcrypt with salt rounds = 10
  • Store user record in database with status "PENDING_VERIFICATION"
  • Generate OTP (6 digits) and store with 15-minute expiry
  • Return response with user ID and message
  • Handle exceptions with appropriate HTTP status codes
  • Log registration attempt with user email

API Specification

{ "method": "POST", "endpoint": "/api/v1/auth/register", "request": { "email": "user@example.com", "phone": "+1234567890", "password": "SecurePass123!", "confirmPassword": "SecurePass123!" }, "response": { "success": true, "message": "Registration successful", "userId": "UUID", "verificationEmailSent": true } }
CUST-INT-001
Email Verification Integration
Integration
SendGrid
Critical
3 pts

Description

After user registration, send verification email with OTP link to confirm email address.

Acceptance Criteria

  • Call SendGrid API to send email

6. Application Tracking Module

6.1 Application Status Tracking

TRACK-FE-001
Application Status Timeline
Frontend
Timeline
Critical
6 pts

Description

As a customer, I want to see the complete status timeline of my application.

Acceptance Criteria

  • Display vertical timeline showing all status changes
  • Show status steps: Submitted → Under Review → Approved/Rejected
  • Highlight current status
  • Show date and time for each status change
  • Show comments from loan officer for each step
  • Show expected timeline (e.g., "Review: 3-5 days")
  • Show next expected action
  • Color-code statuses (green=approved, red=rejected, blue=pending)

Status Workflow

1. SUBMITTED (Customer submits) → Awaiting Documents 2. DOCUMENTS_SUBMITTED → Under Document Review (1-2 days) 3. DOCUMENTS_VERIFIED → Credit Check In Progress (2-3 days) 4. UNDER_REVIEW → Final Review (1-2 days) 5. APPROVED → Offer Sent OR REJECTED → Rejection Reason Provided

CALC-BE-001
Calculate EMI with Amortization
Backend
POST /api/v1/calculator/emi
High
4 pts

Description

Calculate EMI and create complete amortization schedule.

Acceptance Criteria

  • Endpoint: POST /api/v1/calculator/emi
  • Input: principal, rate, tenure
  • Calculate monthly EMI
  • Generate month-by-month amortization
  • Calculate total interest and total payable
  • No authentication required
  • Cache results for 1 hour

EMI Formula

Monthly Rate = Annual Rate / 12 / 100 Number of Payments = Tenure EMI = Principal × (Monthly Rate × (1 + Monthly Rate)^n) / ((1 + Monthly Rate)^n - 1) For each month: Interest = Remaining Principal × Monthly Rate Principal = EMI - Interest Remaining = Remaining - Principal

10. System Integration Points

10.1 Architecture Integration Flow

Frontend (React) ↓ API Gateway (Azure) ↓ Authentication Service (OAuth 2.0/JWT) ↓ Application Services ├── Customer Service ├── Application Service ├── Document Service ├── Notification Service └── Reporting Service ↓ Data Layer ├── SQL Server Database ├── Azure Blob Storage (Documents) └── Redis Cache ↓ External Services ├── SendGrid (Email) ├── Azure Key Vault (Secrets) └── Application Insights (Logging)

10.2 API Gateway Specifications

Base URL: https://api.smartloan.com/api/v1

Authentication: JWT Token in Authorization header: Authorization: Bearer {token}

Rate Limiting: 100 requests per minute per user

Pagination: Default limit: 10, Max limit: 100

10.3 Response Format

Success (2xx): { "success": true, "message": "Operation successful", "data": {}, "timestamp": "2026-05-21T10:30:00Z" } Error (4xx/5xx): { "success": false, "error": "Error message", "code": "ERROR_CODE", "timestamp": "2026-05-21T10:30:00Z" }
